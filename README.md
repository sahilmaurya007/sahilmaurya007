## 🚀 Research-Grade Projects (Portfolio)

### **Project 1: Kyber-Accel-SoC (Secure SoC with PQC Coprocessor)**
*   **Domain**: Post-Quantum Cryptography & RISC-V Architecture.
*   **Problem**: Lattice-based cryptography (LBC) is computationally expensive, causing performance bottlenecks on standard 32-bit RISC-V cores.
*   **Solution**: Designed a hardware NTT (Number Theoretic Transform) accelerator interfaced with a 5-stage RISC-V pipeline via AXI4-Lite.
*   **Key Innovations**: 
    - Multi-banked SRAM for conflict-free coefficient access during butterfly operations.
    - Custom ISA extensions to trigger cryptographic operations directly from C code.
*   **Results**: 12.5x speedup for Kyber-768 Key Encapsulation (KEM) compared to software-only implementation.
*   **Tools**: Vivado 2024.2, OpenLane (Sky130).

### **Project 2: AxMM-RLWE (Approximate Modular Multiplier)**
*   **Domain**: Hardware Security & Low-Power VLSI.
*   **Problem**: Precise modular multiplication consumes significant area and power, limiting its use in energy-harvesting IoT nodes.
*   **Solution**: Implemented an Error-Configurable Approximate Modular Multiplier (AxMM) for Ring-LWE.
*   **Results**: 
    - 22% reduction in LUT usage on Xilinx Kintex-7.
    - 15.4% better Area-Time-Product (ATP) on 45nm Nangate ASIC library.
    - Maintained >99% cryptographic accuracy for R-LWE decryption.
*   **Status**: Published in IEEE SILCON 2024.

### **Project 3: Neuromorphic-RV32 (Adaptive Precision AI Accelerator)**
*   **Domain**: AI Hardware & Neuromorphic Computing.
*   **Problem**: Spiking Neural Networks (SNN) exhibit varying precision requirements during different learning phases.
*   **Solution**: A RISC-V based SoC with a custom "Adaptive Precision Unit" that scales bit-width (4-bit to 32-bit) in real-time based on neuronal state activity.
*   **Results**: 35% reduction in dynamic power consumption during inference.
*   **Status**: Accepted at HiPC 2025.

### **Project 4: Toom-Cook-3 Cryptographic Multiplier**
*   **Domain**: High-Speed Hardware Arithmetic.
*   **Design**: Implementation of 32-bit Toom-Cook-2.5 and Toom-Cook-3 multiplication algorithms for large integer multiplication.
*   **Metrics**: Performed comparative analysis between Toom-Cook and Karatsuba multipliers in terms of frequency, power, and area.

## 🏗️ Advanced Integration: PQC-Integrated RISC-V SoC
<div align="center">
  <img src="https://raw.githubusercontent.com/sahilmaurya007/sahilmaurya007/main/soc_arch.png" width="85%" alt="SoC Architecture Diagram" />
  <br>
  <i>Conceptual Architecture: RV32I Core with PQC Coprocessor via AMBA AXI Interconnect</i>
</div>

**Architectural Interaction**:
1.  **Memory Mapping**:
    - `0x4000_0000`: PQC_CTRL (Bit 0: Start, Bit 1: Reset).
    - `0x4000_0004`: PQC_STAT (Bit 0: Busy, Bit 1: Done).
    - `0x4000_1000 - 0x4000_1FFF`: NTT Data Memory (Shared coefficients).
2.  **Interaction Flow**:
    - RISC-V writes polynomial coefficients to the shared memory.
    - RISC-V triggers the coprocessor by writing to `PQC_CTRL`.
    - PQC Coprocessor performs NTT butterfly operations independently.
    - PQC raises an interrupt on completion; RISC-V reads back the transformed data.
3.  **Implementation**:
    - **FPGA**: Verified on Zynq UltraScale+ ZCU102 using ChipScoPy for real-time monitoring.
    - **ASIC**: Synthesized using Cadence Genus for 28nm FD-SOI technology.

## 🔬 Research Methodology: Side-Channel Analysis Sandbox
My work involves evaluating hardware security through rigorous analysis. The following is a summary of my evaluation pipeline for side-channel resilience.

### **1. Power Analysis Framework**
To evaluate the power leakage of my cryptographic accelerators, I utilize a CAD-based flow:
1. **Simulation**: Running post-synthesis netlist simulations using Xcelium to generate VCD (Value Change Dump) files.
2. **Activity Extraction**: Converting VCD to SAIF (Switching Activity Interchange Format) for specific cryptographic rounds.
3. **Power Profiling**: Using Cadence Joules to map switching activity to cycle-accurate power consumption.
4. **Leakage Modeling**: Applying Differential Power Analysis (DPA) on the power logs to identify correlation between power consumption and secret keys.

### **2. FPGA-Based SCA Evaluation**
On the ZCU102 board, I implement a custom monitor that triggers on cryptographic operations, allowing for synchronized power measurement using external shunts and high-speed ADCs.
